Канонічне URL-посилання

Тег канонічної URL-адреси.Тег канонічного посилання.

Переспрямування тегу канонічного посилання HTML

Канонічне посилання не переспрямовує на бажану URL-адресу, але воно може бути альтернативою переспрямуванню URL-адреси для веб-сайтів, де більшість трафіку надходить із пошукових систем.

Тег HTML для канонічних посилань можна використовувати, коли є кілька сторінок із подібним вмістом і ви хочете повідомити пошуковим системам, якій сторінці ви віддаєте перевагу в результатах пошуку.

Тег канонічного посилання може посилатися як на один домен, так і на кілька доменів .

Додайте тег канонічного посилання на стару сторінку, щоб перейти на
нову сторінку.

Додайте тег канонічного посилання до сторінок, які ви не бажаєте  пов’язувати трафік  пошукової  системи  з бажаною сторінкою.

У розділ потрібно додати тег канонічного посилання.

Приклад №1

Коли сторінку переміщено, ми можемо додати канонічне посилання на стару сторінку, щоб сповістити пошукові системи про необхідність переходу на нову сторінку.

стара сторінка1.html:

<!DOCTYPE html>
<html>
<head>
   ...
   <link rel="canonical" href="http://www.mydomain.com/new-page1.html">
</head>
<body>
   ...
</body>
</html>

Приклад №2

Якщо є кілька сторінок із подібним змістом, якщо ми вважаємо за краще, щоб пошукові системи досягали page3.html, ми повинні додати канонічне посилання на page3.html у головному розділі page1.html і page2.html.

page1.html:

<!DOCTYPE html>
<html>
<head>
   ...
   <link rel="canonical" href="http://www.mydomain.com/page3.html">
</head>
<body>
   ...
</body>
</html>

page2.html:

<!DOCTYPE html>
<html>
<head>
   ...
   <link rel="canonical" href="http://www.mydomain.com/page3.html">
</head>
<body>
   ...
</body>
</html>
