Kanonischer URL-Link

Kanonisches URL-Tag. Canonical-Link-Tag.

HTML-Canonical-Link-Tag-Weiterleitung

Der kanonische Link leitet nicht zur bevorzugten URL weiter, kann aber eine Alternative zur URL-Umleitung für Websites sein, auf denen der meiste Traffic von Suchmaschinen kommt.

Das HTML-Tag für kanonische Links kann verwendet werden, wenn es mehrere Seiten mit ähnlichen Inhalten gibt und Sie Suchmaschinen mitteilen möchten, welche Seite Sie in den Suchergebnissen bevorzugen.

Das kanonische Link-Tag kann auf dieselbe Domain und auch auf mehrere Domains verlinken .

Fügen Sie der alten Seite das kanonische Link-Tag hinzu, um auf die
neue Seite zu verlinken.

Fügen Sie das Tag für kanonische Links zu Seiten hinzu, auf denen Sie den   Suchmaschinenverkehr  nicht mit  der bevorzugten Seite verknüpfen möchten.

Das kanonische Link-Tag sollte im Abschnitt hinzugefügt werden.

Beispiel 1

Wenn die Seite verschoben wurde, können wir einen kanonischen Link auf der alten Seite hinzufügen, um Suchmaschinen zu benachrichtigen, die neue Seite zu erreichen.

alte-seite1.html:

<!DOCTYPE html>
<html>
<head>
   ...
   <link rel="canonical" href="http://www.mydomain.com/new-page1.html">
</head>
<body>
   ...
</body>
</html>

Beispiel #2

Wenn es mehrere Seiten mit ähnlichem Inhalt gibt und wir es Suchmaschinen vorziehen, Seite3.html zu erreichen, sollten wir einen kanonischen Link zu Seite3.html im Head-Bereich von Seite1.html und Seite2.html hinzufügen.

Seite1.html:

<!DOCTYPE html>
<html>
<head>
   ...
   <link rel="canonical" href="http://www.mydomain.com/page3.html">
</head>
<body>
   ...
</body>
</html>

Seite2.html:

<!DOCTYPE html>
<html>
<head>
   ...
   <link rel="canonical" href="http://www.mydomain.com/page3.html">
</head>
<body>
   ...
</body>
</html>
