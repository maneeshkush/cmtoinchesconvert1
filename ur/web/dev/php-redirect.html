پی ایچ پی ری ڈائریکٹ

پی ایچ پی کو صفحہ سے یو آر ایل پر ری ڈائریکٹ کریں۔پی ایچ پی 301 ری ڈائریکٹ۔

اس پی ایچ پی کی ری ڈائریکشن کو HTTP رسپانس اسٹیٹس کوڈ واپس کرنا چاہیے: 301 مستقل طور پر منتقل کر دیا گیا ہے۔

سرچ انجن 301 رسپانس اسٹیٹس کوڈ کا استعمال کرتے ہیں تاکہ پیج رینک کو پرانے یو آر ایل سے نئے یو آر ایل میں منتقل کیا جا سکے۔

پی ایچ پی ہیڈر ری ڈائریکٹ

old-page.php کوڈ کو ری ڈائریکشن کوڈ سے new-page.php پر تبدیل کریں۔

old-page.php:

<?php
// PHP permanent URL redirection
header("Location: http://www.domain.com/new-page.php", true, 301);
exit();
?>

پرانے صفحہ میں .php فائل کی توسیع ہونی چاہیے۔

نیا صفحہ کسی بھی ایکسٹینشن کے ساتھ ہو سکتا ہے۔

پی ایچ پی کی ری ڈائریکٹ مثالیں۔

مثال نمبر 1

php-redirect-test.php

<?php
// PHP permanent URL redirection test
header("Location: https://www.cmtoinchesconvert.com/ur/web/dev/php-redirect.html", true, 301);
exit();
?>

 

php-redirect-test.php سے اس صفحہ پر واپس بھیجنے کے لیے اس لنک کو دبائیں:

 

پی ایچ پی ری ڈائریکٹ ٹیسٹ - پی ایچ پی فائل

مثال نمبر 2

php-redirect-test.htm

<?php
// PHP permanent URL redirection test
header("Location: https://www.cmtoinchesconvert.com/ur/web/dev/php-redirect.html", true, 301);
exit();
?>

 

html فائل php-redirect-test.htm سے پی ایچ پی ری ڈائریکشن عام طور پر .html فائل ایکسٹینشن کی وجہ سے کام نہیں کرے گا، جب تک کہ یہ .htaccess یا httpd.conf فائل میں فعال نہ ہو:

 

پی ایچ پی ری ڈائریکٹ ٹیسٹ - HTML فائل

 

ایچ ٹی ایم ایل فائلوں میں پی ایچ پی کو فعال کرنے کے لیے اس کوڈ کو .htaccess یا httpd.conf فائل میں شامل کریں:

Addtype application/x-httpd-php .htm .html
