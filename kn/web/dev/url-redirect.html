    • ಕ್ಲೈಂಟ್ ಸೈಡ್ ಮರುನಿರ್ದೇಶನ
    • HTTP ಸ್ಥಿತಿ ಕೋಡ್‌ಗಳು

URL ಮರುನಿರ್ದೇಶನ

URL ಪುಟ ಮರುನಿರ್ದೇಶನವು ಒಂದು URL ನಿಂದ ಇನ್ನೊಂದು URL ಗೆ ಸ್ವಯಂಚಾಲಿತ URL ಬದಲಾವಣೆಯ ಕಾರ್ಯಾಚರಣೆಯಾಗಿದೆ.

ಈ ಮರುನಿರ್ದೇಶನವನ್ನು ಈ ಕೆಳಗಿನ ಕಾರಣಗಳಿಗಾಗಿ ಮಾಡಲಾಗಿದೆ:

  1. ಹಳೆಯ ಬಳಕೆಯಲ್ಲಿಲ್ಲದ URL ನಿಂದ ಹೊಸ ನವೀಕರಿಸಿದ URL ಗೆ ಮರುನಿರ್ದೇಶಿಸಿ.
  2. ಹಳೆಯ ಬಳಕೆಯಲ್ಲಿಲ್ಲದ ಡೊಮೇನ್‌ನಿಂದ ಹೊಸ ಡೊಮೇನ್‌ಗೆ ಮರುನಿರ್ದೇಶಿಸಿ.
  3. www ಅಲ್ಲದ ಡೊಮೇನ್ ಹೆಸರಿನಿಂದ www ಡೊಮೇನ್ ಹೆಸರಿಗೆ ಮರುನಿರ್ದೇಶಿಸಿ.
  4. ಚಿಕ್ಕ URL ಹೆಸರಿನಿಂದ ದೀರ್ಘ URL ಹೆಸರಿಗೆ ಮರುನಿರ್ದೇಶಿಸಿ - URL ಸಂಕ್ಷಿಪ್ತಗೊಳಿಸುವ ಸೇವೆ.
  5. URL ಸಂಕ್ಷಿಪ್ತಗೊಳಿಸುವ ಸೇವೆಯು ಬಳಕೆದಾರರಿಗೆ ಚಿಕ್ಕ URL ಅನ್ನು ಸೇರಿಸಲು ಅನುಮತಿಸುತ್ತದೆ ಮತ್ತು ನೈಜ ಪುಟದ ವಿಷಯಗಳನ್ನು ಹೊಂದಿರುವ ದೀರ್ಘ URL ಅನ್ನು ಮರುನಿರ್ದೇಶಿಸುತ್ತದೆ.

ಬಳಕೆದಾರರು ಹಳೆಯ ಬಾಹ್ಯ ಲಿಂಕ್‌ಗಳು ಅಥವಾ ಬುಕ್‌ಮಾರ್ಕ್‌ನಿಂದ ಹಳೆಯ URL ಅನ್ನು ತಲುಪಬಹುದು.

ಸ್ಕ್ರಿಪ್ಟ್ ಅನ್ನು ಸೇರಿಸುವ ಸೈಟ್‌ನ ವೆಬ್‌ಮಾಸ್ಟರ್‌ನಿಂದ.

ಸರ್ವರ್ ಸೈಡ್ ಮರುನಿರ್ದೇಶನ

Apache / IIS ಸರ್ವರ್ ಸಾಫ್ಟ್‌ವೇರ್ ಅನ್ನು ಕಾನ್ಫಿಗರ್ ಮಾಡುವ ಮೂಲಕ ಅಥವಾ PHP / ASP / ASP.NET ಸ್ಕ್ರಿಪ್ಟ್ ಬಳಸುವ ಮೂಲಕ ಸರ್ವರ್‌ನಲ್ಲಿ ಸರ್ವರ್ ಸೈಡ್ ಮರುನಿರ್ದೇಶನವನ್ನು ಮಾಡಲಾಗುತ್ತದೆ.

URL ಗಳನ್ನು ಮರುನಿರ್ದೇಶಿಸಲು ಇದು ಆದ್ಯತೆಯ ಮಾರ್ಗವಾಗಿದೆ, ಏಕೆಂದರೆ ನೀವು HTTP 301 ಸರಿಸಿದ ಶಾಶ್ವತ ಸ್ಥಿತಿ ಕೋಡ್ ಅನ್ನು ಹಿಂತಿರುಗಿಸಬಹುದು.

ಹಳೆಯ URL ನಿಂದ ಹೊಸ URL ಗೆ ಪುಟ ಶ್ರೇಣಿಯನ್ನು ವರ್ಗಾಯಿಸಲು ಹುಡುಕಾಟ ಇಂಜಿನ್‌ಗಳು 301 ಸ್ಥಿತಿಯನ್ನು ಬಳಸುತ್ತವೆ.

ಕ್ಲೈಂಟ್ ಸೈಡ್ ಮರುನಿರ್ದೇಶನ

HTML ಮೆಟಾ ರಿಫ್ರೆಶ್ ಟ್ಯಾಗ್ ಅಥವಾ ಜಾವಾಸ್ಕ್ರಿಪ್ಟ್ ಕೋಡ್ ಅನ್ನು ಬಳಸಿಕೊಂಡು ಬಳಕೆದಾರರ ವೆಬ್ ಬ್ರೌಸರ್‌ನಲ್ಲಿ ಕ್ಲೈಂಟ್ ಸೈಡ್ ಮರುನಿರ್ದೇಶನವನ್ನು ಮಾಡಲಾಗುತ್ತದೆ.

HTTP 301 ಮರುನಿರ್ದೇಶನ

HTTP 301 ಸರಿಸಲಾಗಿದೆ ಶಾಶ್ವತವಾಗಿ ಸ್ಥಿತಿ ಕೋಡ್ ಎಂದರೆ ಶಾಶ್ವತ URL ಮರುನಿರ್ದೇಶನ.

301 ಮರುನಿರ್ದೇಶನವು URL ಗಳನ್ನು ಮರುನಿರ್ದೇಶಿಸಲು ಆದ್ಯತೆಯ ಮಾರ್ಗವಾಗಿದೆ, ಏಕೆಂದರೆ ಇದು URL ಉತ್ತಮವಾಗಿದೆ ಎಂದು ಸರ್ಚ್ ಇಂಜಿನ್‌ಗಳಿಗೆ ತಿಳಿಸುತ್ತದೆ ಮತ್ತು ಹುಡುಕಾಟ ಎಂಜಿನ್‌ಗಳು ಹಳೆಯ URL ಪುಟದ ಬದಲಿಗೆ ಹೊಸ URL ಪುಟವನ್ನು ಹುಡುಕಾಟ ಫಲಿತಾಂಶಗಳಲ್ಲಿ ಇರಿಸಬೇಕು ಮತ್ತು ಹೊಸ URL ಪುಟವನ್ನು ವರ್ಗಾಯಿಸಬೇಕು, ಹಳೆಯ URL ಪುಟದ ಪುಟ ಶ್ರೇಣಿ.

301 ಮರುನಿರ್ದೇಶನವನ್ನು ಡೊಮೇನ್‌ಗಳಾದ್ಯಂತ ಅಥವಾ ಅದೇ ಡೊಮೇನ್‌ನಲ್ಲಿ ಮಾಡಬಹುದು.

301 ಮರುನಿರ್ದೇಶನವನ್ನು ಬಳಸಲುGoogle ಶಿಫಾರಸು ಮಾಡುತ್ತದೆ.

ಮರುನಿರ್ದೇಶನ ಆಯ್ಕೆಗಳು

ಸ್ಕ್ರಿಪ್ಟ್ ಮರುನಿರ್ದೇಶನ ಮರುನಿರ್ದೇಶನ ಬದಿ ಹಳೆಯ ಪುಟದ ಫೈಲ್ ಪ್ರಕಾರ ಮರುನಿರ್ದೇಶನ URL ಅಥವಾ ಡೊಮೇನ್ ಹಳೆಯ URL ಸರ್ವರ್ ಪ್ರಕಾರ 301 ಮರುನಿರ್ದೇಶನ ಬೆಂಬಲ
PHP ಸರ್ವರ್-ಸೈಡ್ .php URL ಅಪಾಚೆ / ಲಿನಕ್ಸ್ ಹೌದು
ಎಎಸ್ಪಿ ಸರ್ವರ್-ಸೈಡ್ .asp URL IIS / ವಿಂಡೋಸ್ ಹೌದು
ASP.NET ಸರ್ವರ್-ಸೈಡ್ .aspx URL IIS / ವಿಂಡೋಸ್ ಹೌದು
.htaccess ಸರ್ವರ್-ಸೈಡ್ ಎಲ್ಲಾ URL / ಡೊಮೇನ್ ಅಪಾಚೆ / ಲಿನಕ್ಸ್ ಹೌದು
IIS ಸರ್ವರ್-ಸೈಡ್ ಎಲ್ಲಾ URL / ಡೊಮೇನ್ IIS / ವಿಂಡೋಸ್ ಹೌದು
HTML ಅಂಗೀಕೃತ ಲಿಂಕ್ ಟ್ಯಾಗ್ ಕ್ಲೈಂಟ್-ಸೈಡ್ .html URL ಎಲ್ಲಾ ಇಲ್ಲ
HTML ಮೆಟಾ ರಿಫ್ರೆಶ್ ಕ್ಲೈಂಟ್-ಸೈಡ್ .html URL ಎಲ್ಲಾ ಇಲ್ಲ
HTML ಫ್ರೇಮ್ ಕ್ಲೈಂಟ್-ಸೈಡ್ .html URL ಎಲ್ಲಾ ಇಲ್ಲ
ಜಾವಾಸ್ಕ್ರಿಪ್ಟ್ ಕ್ಲೈಂಟ್-ಸೈಡ್ .html URL ಎಲ್ಲಾ ಇಲ್ಲ
jQuery ಕ್ಲೈಂಟ್-ಸೈಡ್ .html URL ಎಲ್ಲಾ ಇಲ್ಲ

ಮರುನಿರ್ದೇಶನ ಸ್ಕ್ರಿಪ್ಟ್ - ಮರುನಿರ್ದೇಶನಕ್ಕಾಗಿ ಬಳಸಲಾಗುವ ಸ್ಕ್ರಿಪ್ಟಿಂಗ್ ಭಾಷೆ.

ಮರುನಿರ್ದೇಶನ ಸೈಡ್ - ಅಲ್ಲಿ ಮರುನಿರ್ದೇಶನ ನಡೆಯುತ್ತದೆ - ಸರ್ವರ್-ಸೈಡ್ ಅಥವಾ ಕ್ಲೈಂಟ್-ಸೈಡ್ .

ಹಳೆಯ ಪುಟ ಫೈಲ್ ಪ್ರಕಾರ - ಮರುನಿರ್ದೇಶನ ಕೋಡ್‌ನ ಸ್ಕ್ರಿಪ್ಟಿಂಗ್ ಭಾಷೆಯನ್ನು ಒಳಗೊಂಡಿರುವ ಹಳೆಯ URL ಪುಟದ ಪ್ರಕಾರ.

ಮರುನಿರ್ದೇಶನ URL ಅಥವಾ ಡೊಮೇನ್ -ಒಂದೇ ವೆಬ್ ಪುಟದ URL ಮರುನಿರ್ದೇಶನ ಅಥವಾ ಸಂಪೂರ್ಣ ವೆಬ್‌ಸೈಟ್‌ನ ಡೊಮೇನ್ ಮರುನಿರ್ದೇಶನವನ್ನು ಬೆಂಬಲಿಸುತ್ತದೆ.

ವಿಶಿಷ್ಟವಾದ ಹಳೆಯ URL ಸರ್ವರ್ ಪ್ರಕಾರ - ಸರ್ವರ್‌ನ ವಿಶಿಷ್ಟ ಸಾಫ್ಟ್‌ವೇರ್ ಮತ್ತು ಆಪರೇಟಿಂಗ್ ಸಿಸ್ಟಮ್.

301 ಮರುನಿರ್ದೇಶನ ಬೆಂಬಲ - ಶಾಶ್ವತ 301 ಮರುನಿರ್ದೇಶನ ಸ್ಥಿತಿಯ ಪ್ರತಿಕ್ರಿಯೆಯನ್ನು ಹಿಂತಿರುಗಿಸಬಹುದೇ ಎಂದು ಸೂಚಿಸುತ್ತದೆ.

PHP ಮರುನಿರ್ದೇಶನ

old-page.php ಕೋಡ್ ಅನ್ನು ಹೊಸ-page.php ಗೆ ಮರುನಿರ್ದೇಶನ ಕೋಡ್‌ನೊಂದಿಗೆ ಬದಲಾಯಿಸಿ.

old_page.php:

<?php
// PHP permanent URL redirection
header("Location: http://www.mydomain.com/new-page.php", true, 301);
exit();
?>

ಹಳೆಯ ಪುಟವು .php ಫೈಲ್ ವಿಸ್ತರಣೆಯನ್ನು ಹೊಂದಿರಬೇಕು.

ಹೊಸ ಪುಟವು ಯಾವುದೇ ವಿಸ್ತರಣೆಯೊಂದಿಗೆ ಇರಬಹುದು.

ನೋಡಿ: PHP ಮರುನಿರ್ದೇಶನ

Apache .htaccess ಮರುನಿರ್ದೇಶನ

.htaccess ಫೈಲ್ ಅಪಾಚೆ ಸರ್ವರ್‌ನ ಸ್ಥಳೀಯ ಕಾನ್ಫಿಗರೇಶನ್ ಫೈಲ್ ಆಗಿದೆ.

ನೀವು httpd.conf ಫೈಲ್ ಅನ್ನು ಬದಲಾಯಿಸಲು ಅನುಮತಿಯನ್ನು ಹೊಂದಿದ್ದರೆ, .htaccess ಫೈಲ್ಬದಲಿಗೆ httpd.conf ನಲ್ಲಿ ಮರುನಿರ್ದೇಶನ ನಿರ್ದೇಶನವನ್ನು ಸೇರಿಸುವುದು ಉತ್ತಮ .

ಏಕ URL ಮರುನಿರ್ದೇಶನ

old-page.html ನಿಂದ new-page.html ಗೆ ಶಾಶ್ವತ ಮರುನಿರ್ದೇಶನ.

.htaccess:

Redirect 301 /old-page.html http://www.mydomain.com/new-page.html

ಸಂಪೂರ್ಣ ಡೊಮೇನ್ ಮರುನಿರ್ದೇಶನ

ಎಲ್ಲಾ ಡೊಮೇನ್ ಪುಟಗಳಿಂದ newdomain.com ಗೆ ಶಾಶ್ವತ ಮರುನಿರ್ದೇಶನ.

 .htaccess ಫೈಲ್ ಹಳೆಯ ವೆಬ್‌ಸೈಟ್‌ನ ಮೂಲ ಡೈರೆಕ್ಟರಿಯಲ್ಲಿರಬೇಕು.

.htaccess:

Redirect 301 / http://www.newdomain.com/

ನೋಡಿ: .htaccess ಮರುನಿರ್ದೇಶನ

ASP ಮರುನಿರ್ದೇಶನ

old-page.asp:

<%@ Language="VBScript" %>
<%
' ASP permanent URL redirection
Response.Status="301 Moved Permanently"
Response.AddHeader "Location", "http://www.mydomain.com/new-page.html"
Response.End
%>

ASP.NET ಮರುನಿರ್ದೇಶನ

old-page.aspx:

<script language="C#" runat="server">
// ASP.net permanent URL redirection
private void Page_Load(object sender, EventArgs e)
{
   Response.Status = "301 Moved Permanently";
   Response.AddHeader("Location","http://www.mydomain.com/new-page.html");
   Response.End();
}
</script>

HTML ಮೆಟಾ ರಿಫ್ರೆಶ್ ಮರುನಿರ್ದೇಶನ

HTML ಮೆಟಾ ರಿಫ್ರೆಶ್ ಟ್ಯಾಗ್ ಮರುನಿರ್ದೇಶನವು 301 ಶಾಶ್ವತ ಮರುನಿರ್ದೇಶನ ಸ್ಥಿತಿ ಕೋಡ್ ಅನ್ನು ಹಿಂತಿರುಗಿಸುವುದಿಲ್ಲ, ಆದರೆ Google ನಿಂದ 301 ಮರುನಿರ್ದೇಶನ ಎಂದು ಪರಿಗಣಿಸಲಾಗಿದೆ.

ನೀವು ಮರುನಿರ್ದೇಶಿಸಲು ಬಯಸುವ ಪುಟದ URL ನೊಂದಿಗೆ ಮರುನಿರ್ದೇಶನ ಕೋಡ್‌ನೊಂದಿಗೆ ಹಳೆಯ ಪುಟವನ್ನು ಬದಲಾಯಿಸಿ.

old-page.html:

<!-- HTML meta refresh URL redirection -->
<html>
<head>
   <meta http-equiv="refresh"
   content="0; url=http://www.mydomain.com/new-page.html">
</head>
<body>
   <p>The page has moved to:
   <a href="http://www.mydomain.com/new-page.html">this page</a></p>
</body>
</html>

HTML ಅಂಗೀಕೃತ ಲಿಂಕ್ ಟ್ಯಾಗ್ ಮರುನಿರ್ದೇಶನ

ಅಂಗೀಕೃತ ಲಿಂಕ್ ಪೂರ್ವನಿರ್ದೇಶಿತ URL ಗೆ ಮರುನಿರ್ದೇಶಿಸುವುದಿಲ್ಲ, ಆದರೆ ಹೆಚ್ಚಿನ ಟ್ರಾಫಿಕ್ ಸರ್ಚ್ ಇಂಜಿನ್‌ಗಳಿಂದ ಬರುವ ವೆಬ್‌ಸೈಟ್‌ಗಳಿಗೆ URL ಮರುನಿರ್ದೇಶನಕ್ಕೆ ಪರ್ಯಾಯವಾಗಿರಬಹುದು.

ಒಂದೇ ರೀತಿಯ ವಿಷಯದೊಂದಿಗೆ ಹಲವಾರು ಪುಟಗಳಿರುವಾಗ HTML ಅಂಗೀಕೃತ ಲಿಂಕ್ ಟ್ಯಾಗ್ ಅನ್ನು ಬಳಸಬಹುದು ಮತ್ತು ಹುಡುಕಾಟ ಫಲಿತಾಂಶಗಳಲ್ಲಿ ನೀವು ಯಾವ ಪುಟವನ್ನು ಬಳಸಲು ಬಯಸುತ್ತೀರಿ ಎಂಬುದನ್ನು ಹುಡುಕಾಟ ಎಂಜಿನ್‌ಗಳಿಗೆ ಹೇಳಲು ನೀವು ಬಯಸುತ್ತೀರಿ.

ಅಂಗೀಕೃತ ಲಿಂಕ್ ಟ್ಯಾಗ್ ಒಂದೇ ಡೊಮೇನ್ ಮತ್ತು ಕ್ರಾಸ್ ಡೊಮೇನ್‌ಗೆ ಲಿಂಕ್ ಮಾಡಬಹುದು.

ಹೊಸ ಪುಟಕ್ಕೆ ಲಿಂಕ್ ಮಾಡಲು ಹಳೆಯ ಪುಟಕ್ಕೆ ಅಂಗೀಕೃತ ಲಿಂಕ್ ಟ್ಯಾಗ್ ಅನ್ನು ಸೇರಿಸಿ.

ಆದ್ಯತೆಯ ಪುಟಕ್ಕೆ ಲಿಂಕ್ ಮಾಡಲು ಸರ್ಚ್ ಇಂಜಿನ್ ದಟ್ಟಣೆಯನ್ನು ಪಡೆಯದಿರಲು ನೀವು ಆದ್ಯತೆ ನೀಡುವ ಪುಟಗಳಿಗೆ ಅಂಗೀಕೃತ ಲಿಂಕ್ ಟ್ಯಾಗ್ ಅನ್ನು ಸೇರಿಸಿ.

ಅಂಗೀಕೃತ ಲಿಂಕ್ ಟ್ಯಾಗ್ ಅನ್ನು <head> ವಿಭಾಗದಲ್ಲಿ ಸೇರಿಸಬೇಕು.

old-page.html:

<link rel="canonical" href="http://www.mydomain.com/new-page.html">

ನೋಡಿ: ಅಂಗೀಕೃತ URL ಲಿಂಕ್

HTML ಫ್ರೇಮ್ ಮರುನಿರ್ದೇಶನ

ಫ್ರೇಮ್ ಮರುನಿರ್ದೇಶನದಲ್ಲಿ new-page.html ಫೈಲ್ ಅನ್ನು html ಫ್ರೇಮ್ ಮೂಲಕ ವೀಕ್ಷಿಸಲಾಗುತ್ತದೆ.

ಇದು ನಿಜವಾದ URL ಮರುನಿರ್ದೇಶನವಲ್ಲ.

ಫ್ರೇಮ್ ಮರುನಿರ್ದೇಶನವು ಹುಡುಕಾಟ ಎಂಜಿನ್ ಸ್ನೇಹಿಯಾಗಿಲ್ಲ ಮತ್ತು ಶಿಫಾರಸು ಮಾಡಲಾಗಿಲ್ಲ.

old-page.html:

<!-- HTML frame redirection -->
<html>
<head>
    <title>Title of new page</title>
</head>
<frameset cols="100%">
    <frame src="http://www.mydomain.com/new-page.html">
    <noframes>
     <a href="http://www.mydomain.com/new-page.html">Link to new page</a>
    </noframes>
</frameset>
</html>
